Read the complete user request before editing.

Before reporting completion, verify every checklist item against the actual implementation.

When requirements conflict with the existing code, follow the new requirement unless preserving something was explicitly requested.

## Visual Standard

The website must feel:

- Premium
- Editorial
- Tactile
- Physical
- Confident
- Spacious
- Carefully art-directed

It must not feel like:

- A generic Tailwind template
- A SaaS landing page
- An AI-generated website
- A cheap local-business template
- A collection of identical cards
- A website covered in tight outlined buttons
- An ecommerce theme

## Design Rules

- Use intentional visual hierarchy.
- Give typography enough breathing room.
- Do not place borders tightly around button text.
- Use generous but controlled button padding.
- Do not use pill shapes everywhere.
- Do not use excessive rounded cards.
- Do not repeat heading, paragraph, and three-card layouts throughout the site.
- Do not create unexplained empty gaps.
- Do not treat mobile as desktop stacked vertically.
- Do not use images that do not clearly represent the stated service.
- Do not use random image collages.
- Do not use generic office or city photography for printing services.
- Do not invent testimonials, statistics, awards, customers, or company history.
- Do not retain weak copy simply because it exists.
- Do not add generic marketing cliches.

## Images

Every image must clearly support the surrounding service or content.

Examples:

- Signage must show fascia signs, illuminated signs, acrylic letters, storefront graphics, or installation.
- Clothing must show screen printing, heat pressing, embroidery, or finished branded workwear.
- Vehicle graphics must clearly show vinyl application or finished branded vehicles.
- Window graphics must show installation or completed window vinyl.
- Business print must show cards, flyers, brochures, paper finishes, or production.
- Banners must show banner production, installation, or finished commercial banners.

Use legally reusable temporary images or local placeholders. Do not scrape or hotlink competitor photography.

## Engineering

- Preserve maintainable project structure.
- Use reusable components where repetition is real.
- Do not force visually different sections into one generic component.
- Remove obsolete and conflicting styles.
- Maintain accessibility and keyboard navigation.
- Respect prefers-reduced-motion.
- Avoid layout shifts and horizontal overflow.
- Test at mobile, tablet, laptop, and large-desktop widths.
- Run formatting, linting, type checking, and production builds before completion.
- Review the actual rendered pages, not only the source code.

## Completion Format

At the end of every substantial task, report:

1. Files changed
2. What was implemented
3. What was visually inspected
4. Commands and checks run
5. Any remaining factual placeholders
6. Any requirement that could not be completed and why

Never state that everything was completed when something was skipped.
